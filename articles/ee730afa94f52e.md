ベンチマークの測り方が間違っていました。
response_modelも高速化の対象になるそうです。

間違った情報を掲載してしまい大変申し訳ありません🙇‍♂️
検証終わり次第ベンチマーク部分を更新します。

# はじめに
2月23日にFastAPI 0.131.0がリリースされました。

https://twitter.com/FastAPI/status/2025627146866073881

中でも注目されたのが以下のツイートです。

https://twitter.com/FastAPI/status/2025613852394529114

要約すると、「戻り値に型だけ指定することで、レスポンスの速さが最大2倍になる」とのことです。

え？すごくないすか？

```python
class User(BaseModel):
    id: int
    name: str
    email: str
    is_active: bool

@app.get("/user")
def get_users():
    ...
```

これを...

```python
class User(BaseModel):
    id: int
    name: str
    email: str
    is_active: bool

@app.get("/user", response_model=list[User])
def get_users() -> list[User]:
    ...
```

または

```python
class User(BaseModel):
    id: int
    name: str
    email: str
    is_active: bool

@app.get("/user", response_model=list[User])
def get_users():
    ...
```

こうするだけでパフォーマンスが上がるらしいです。

# なんで早くなったん？

以下のPRを読み解きます。

https://github.com/fastapi/fastapi/pull/14962

```
✨ Serialize JSON response with Pydantic (in Rust), when there's a Pydantic return type or response model

訳:Pydantic の戻り値型またはレスポンスモデルがある場合、Pydantic (Rust 内) を使用して JSON レスポンスをシリアル化します
```

めっちゃシンプルな説明ですね。
PydanticはPythonで使われているバリデーションライブラリで、主にAPI周りの入出力の型チェックに使われています。

PydanticでRust?と思われますが、実は現行のPydantic v2はRustで書かれており、v1に比べて倍以上の処理速度を実現しているとのこと。

早くなっただけでなく、v2ではデフォルトでjsonシリアライズに対応するようになり、Pydanticのみでシリアライズが可能になったそうです。
つまりRust側でjsonシリアライズすることで高速になったそうです。

:::details もうちょい補足

大きく変わったのはPRのこの部分。

https://github.com/fastapi/fastapi/blob/880ad19087c65436da083118aad4b4b9f4da1503/fastapi/routing.py#L290-L291

dump_jsonが有効の場合、serialize_jsonを使ってjsonシリアライズをします。これがRustでシリアライズする関数になります。

https://github.com/fastapi/fastapi/blob/880ad19087c65436da083118aad4b4b9f4da1503/fastapi/_compat/v2.py#L202-L226

一方でdump_jsonが無効の場合は従来通りserializeが使われます。

https://github.com/fastapi/fastapi/blob/880ad19087c65436da083118aad4b4b9f4da1503/fastapi/_compat/v2.py#L177-L200

dump_pythonはpythonオブジェクトを返却するため、それをjsonに変換する手間がありました。
一応変換自体はRustでしているのですが、そこにPythonでのjsonシリアライズが加わるので若干遅くなっていたそうです。

Rustのパフォーマンスに加えて、pythonを経由せずにjsonシリアライズすることになり高速になったそうです。
:::

# ベンチマーク
**見せてもらおうか、新しいPydanticの性能とやらを！**
ということでベンチマークで速くなったのか検証します。

## 方法
以下の2つのエンドポイントを作成し、負荷テストで検証したいと思います。
- 型ヒントなし
- `response_model`を使う方法
- 戻り値の型ヒント（Return Type）を使う方法

大量のデータを返すシナリオが最も差が出やすいため、1,000個のアイテムを返す設定で比較します。(どんなエンドポイントや)

```python
from fastapi import FastAPI
from pydantic import BaseModel

app = FastAPI()

class User(BaseModel):
    id: int
    name: str
    email: str
    is_active: bool

# ダミーデータ1,000件生成
test_data: list[User] = [
    User(
        id=i,
        name=f"User{i}",
        email=f"user{i}@example.com",
        is_active=i % 2 == 0
    )
    for i in range(1000)
]

@app.get("/untyped")
def get_users_old_way():
    """型ヒントなし"""
    return test_data

@app.get("/typed")
def get_users_new_way() -> list[User]:
    """戻り値型ヒントを使用"""
    return test_data

@app.get("/response-model", response_model=list[User])
def get_users_response_model():
    """response_modelを使用"""
    return test_data


@app.get("/health")
def health_check():
    """ヘルスチェック用"""
    return {"status": "ok"}

```

### 100回〜500回 程度のリクエスト

以下のような計測ツールで検証します。

:::details ベンチマークスクリプト
```python
import asyncio
import time
import statistics
import sys
import httpx
from typing import List

async def benchmark_endpoint(client: httpx.AsyncClient, url: str, requests_count: int) -> dict:
    """エンドポイントのベンチマークを実行"""
    response_times = []
    
    print(f"\n🔄 Testing {url} with {requests_count} requests...")
    
    # ウォームアップ（最初の数回は除外）
    for _ in range(5):
        await client.get(url)
    
    start_time = time.perf_counter()
    
    for i in range(requests_count):
        request_start = time.perf_counter()
        response = await client.get(url)
        request_end = time.perf_counter()
        
        if response.status_code != 200:
            print(f"❌ Error: Status code {response.status_code}")
            continue
            
        response_times.append((request_end - request_start) * 1000)  # ms
        
        if (i + 1) % 100 == 0:
            print(f"  Progress: {i + 1}/{requests_count} requests completed")
    
    total_time = time.perf_counter() - start_time
    
    return {
        "url": url,
        "total_requests": requests_count,
        "total_time": total_time,
        "avg_response_time": statistics.mean(response_times),
        "median_response_time": statistics.median(response_times),
        "min_response_time": min(response_times),
        "max_response_time": max(response_times),
        "rps": requests_count / total_time,
        "stdev": statistics.stdev(response_times) if len(response_times) > 1 else 0
    }

async def run_benchmarks(base_url: str, requests_count: int):
    """すべてのベンチマークを実行"""
    async with httpx.AsyncClient(base_url=base_url, timeout=30.0) as client:
        # ヘルスチェック
        print("🏥 Health check...")
        health_response = await client.get("/health")
        if health_response.status_code != 200:
            print("❌ Server is not healthy!")
            sys.exit(1)
        print("✅ Server is healthy\n")
        
        # ベンチマーク実行
        results = []
        
        # Untyped (型指定なし)
        untyped_result = await benchmark_endpoint(client, "/untyped", requests_count)
        results.append(("Untyped (no annotation)", untyped_result))
        
        # 少し待機
        await asyncio.sleep(1)
        
        # Typed (戻り値型ヒント)
        typed_result = await benchmark_endpoint(client, "/typed", requests_count)
        results.append(("Typed (return annotation)", typed_result))
        
        # 少し待機
        await asyncio.sleep(1)
        
        # Response model
        response_model_result = await benchmark_endpoint(client, "/response-model", requests_count)
        results.append(("Response Model", response_model_result))
        
        return results

def print_report(results: List[tuple], python_version: str, fastapi_version: str, pydantic_version: str):
    """結果レポートを出力"""
    print("\n" + "="*80)
    print("📊 BENCHMARK REPORT")
    print("="*80)
    
    print("\n【実行環境】")
    print(f"  Python:  {python_version}")
    print(f"  FastAPI: {fastapi_version}")
    print(f"  Pydantic: {pydantic_version}")
    
    print("\n【比較表】")
    print("-"*80)
    print(f"{'Method':<25} {'Avg (ms)':<12} {'Median (ms)':<12} {'RPS':<12} {'Stdev (ms)':<12}")
    print("-"*80)
    
    for name, result in results:
        print(f"{name:<25} {result['avg_response_time']:>10.2f}  "
              f"{result['median_response_time']:>10.2f}  "
              f"{result['rps']:>10.2f}  "
              f"{result['stdev']:>10.2f}")
    
    print("-"*80)
    
    # 比較分析
    untyped = results[0][1]
    typed = results[1][1]
    response_model = results[2][1]
    
    print("\n【考察】")
    print("-"*80)
    
    # 最速を特定
    fastest_idx = min(range(len(results)), key=lambda i: results[i][1]['avg_response_time'])
    fastest_name = results[fastest_idx][0]
    
    print(f"🏆 最速: {fastest_name}\n")
    
    # Untyped vs Typed
    untyped_vs_typed = ((untyped['avg_response_time'] - typed['avg_response_time']) 
                        / untyped['avg_response_time'] * 100)
    print(f"📊 Untyped vs Typed (return annotation):")
    if untyped_vs_typed > 0:
        print(f"   ✅ Typed の方が {untyped_vs_typed:.2f}% 高速")
    else:
        print(f"   ⚠️  Untyped の方が {abs(untyped_vs_typed):.2f}% 高速")
    print(f"   {untyped['avg_response_time']:.2f}ms → {typed['avg_response_time']:.2f}ms")
    print(f"   RPS: {untyped['rps']:.2f} → {typed['rps']:.2f} ({((typed['rps'] - untyped['rps']) / untyped['rps'] * 100):+.2f}%)")
    
    # Untyped vs Response Model
    untyped_vs_rm = ((untyped['avg_response_time'] - response_model['avg_response_time']) 
                     / untyped['avg_response_time'] * 100)
    print(f"\n📊 Untyped vs Response Model:")
    if untyped_vs_rm > 0:
        print(f"   ✅ Response Model の方が {untyped_vs_rm:.2f}% 高速")
    else:
        print(f"   ⚠️  Untyped の方が {abs(untyped_vs_rm):.2f}% 高速")
    print(f"   {untyped['avg_response_time']:.2f}ms → {response_model['avg_response_time']:.2f}ms")
    print(f"   RPS: {untyped['rps']:.2f} → {response_model['rps']:.2f} ({((response_model['rps'] - untyped['rps']) / untyped['rps'] * 100):+.2f}%)")
    
    # Typed vs Response Model
    typed_vs_rm = ((typed['avg_response_time'] - response_model['avg_response_time']) 
                   / typed['avg_response_time'] * 100)
    print(f"\n📊 Typed vs Response Model:")
    if typed_vs_rm > 0:
        print(f"   ✅ Response Model の方が {typed_vs_rm:.2f}% 高速")
    else:
        print(f"   ⚠️  Typed の方が {abs(typed_vs_rm):.2f}% 高速")
    print(f"   {typed['avg_response_time']:.2f}ms → {response_model['avg_response_time']:.2f}ms")
    print(f"   RPS: {typed['rps']:.2f} → {response_model['rps']:.2f} ({((response_model['rps'] - typed['rps']) / typed['rps'] * 100):+.2f}%)")
    
    print("\n💡 結論:")
    if abs(typed_vs_rm) < 5:
        print("   Typed (戻り値型ヒント) と Response Model のパフォーマンス差は微小（5%未満）。")
        print("   型ヒントの方がPythonらしい記述ができるため、可読性の観点から推奨されます。")
    elif typed_vs_rm < -5:
        print("   Typed (戻り値型ヒント) が Response Model より明確に高速です。")
    else:
        print("   Response Model が Typed より明確に高速です。")
    
    if abs(untyped_vs_typed) > 5 or abs(untyped_vs_rm) > 5:
        print("   型指定（Typed or Response Model）を行うことでパフォーマンスへの影響があります。")
    
    print("="*80)
    print()

async def main():
    """メイン処理"""
    import fastapi
    import pydantic
    
    # バージョン情報取得
    python_version = sys.version.split()[0]
    fastapi_version = fastapi.__version__
    pydantic_version = pydantic.__version__
    
    print("="*80)
    print("🚀 FastAPI Performance Benchmark")
    print("   Untyped vs Typed (return annotation) vs Response Model")
    print("="*80)
    
    base_url = "http://127.0.0.1:8000"
    requests_count = 500  # リクエスト数
    
    print(f"\n⚙️  Configuration:")
    print(f"   Server: {base_url}")
    print(f"   Requests per endpoint: {requests_count}")
    
    # ベンチマーク実行
    results = await run_benchmarks(base_url, requests_count)
    
    # レポート出力
    print_report(results, python_version, fastapi_version, pydantic_version)


if __name__ == "__main__":
    asyncio.run(main())

```
